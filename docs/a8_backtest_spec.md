# A9r2統合バックテスト

## 正式な起動方法

```powershell
python backtest_a9r2.py
```

正式入口は `backtest_a9r2.py` とする。

一度選択した監視銘柄ファイルは `.a8_settings.json` に記憶する。次回起動時にファイルが存在すれば自動入力し、存在しない、または設定を読み込めない場合はファイル選択画面を表示する。

## GUI設定

- 開始日、終了日
- 監視銘柄Markdownファイル
- 出力先フォルダ
- 25日乖離率の最低値、最高値
- VWAP維持確認あり／なし
- エントリー時刻
  - VWAP維持確認あり: 前日終値、11:00、14:00
  - VWAP維持確認なし: 11:00、14:00
- 直近3営業日の安値切り下げ除外回数: 0回、1回、2回、3回
- 直近3営業日の高値更新条件: 考慮しない、1回以上、2回以上、3回
- 終端位置: 考慮せず、30%以上、40%以上、50%以上、60%以上
- 5日線傾き > 0 を条件にする
- 5日線傾き鈍化: 考慮しない、前日・3日前とも許容しない、前日・3日前のいずれかのみ許容、3日前のみ許容、前日のみ許容

開始日の初期値は、実行日から60カレンダー日以内で一番早い営業日とする。終了日の初期値は直近の営業日とする。営業日の初期値計算は土日を除く平日基準で行う。

安値切り下げは、直近3営業日それぞれの安値を前営業日と比較して数える。0回は条件を無効化し、1～3回は実際の回数が選択値以上の場合に除外する。

高値更新は、直近3営業日それぞれの高値を前営業日と比較して数える。考慮しない場合は条件を無効化し、1～3回を選択した場合は実際の回数が選択値未満の場合に除外する。

終端位置は、エントリー時点の価格が当日レンジのどこにあるかを表す。計算式は `(エントリー価格 - 安値) / (高値 - 安値) * 100` とする。11:00、14:00エントリーでは、エントリー時点までに確定した分足の高値～安値を使う。前日終値エントリーでは、前日の日足高値～安値を使う。終端位置を考慮する場合は分足データの取得制約を受けるため、画面の初期期間と同じく直近40営業日程度を対象にする。終端位置を「考慮せず」にした場合、この条件では除外しない。

5日線傾き鈍化は、エントリー判定日の5日線傾きと、前営業日・3営業日前の5日線傾きを比較する。Aを「当日傾き < 前日傾き」、Bを「当日傾き < 3営業日前傾き」とする。「前日・3日前とも許容しない」はAまたはBがあれば除外する。「前日・3日前のいずれかのみ許容」はAとBのどちらか一方のみなら許容し、AとBが両方あれば除外する。「3日前のみ許容」はBのみ許容し、Aがあれば除外する。「前日のみ許容」はAのみ許容し、Bがあれば除外する。「考慮しない」はこの条件では除外しない。

## エントリー条件

次の条件をすべて満たす場合にエントリーする。

- 前日と判定時点の25日乖離率が設定範囲内
- 25日線が横ばい以上
- 5日線傾き条件を有効にした場合、判定時点の5日線が前営業日の5日線より上
- 5日線傾き鈍化を考慮する場合、選択した鈍化許容ルールに抵触しない
- VWAP維持確認ありの場合、判定価格が累積VWAP以上
- VWAP維持確認なしの場合、VWAPとの位置関係にかかわらず指定時刻にエントリー
- 安値切り下げ回数が除外基準未満、または除外設定が0回
- 高値更新回数が必要回数以上、または高値更新条件が考慮しない
- 終端位置を考慮する場合、終端位置が選択した閾値以上

終端位置の計算に必要な高値・安値が取得できない場合、または高値と安値が同値でレンジが0の場合は、終端位置を計算できないためスキップする。

## 結果解析

主項目として5・10・15営業日の各期間について、次を集計する。

- +5%到達率
- -3%到達率
- 最大順行幅（MFE）中央値
- 最大逆行幅（MAE）中央値
- +5%先着率
- -3%先着率

+5%先着率と-3%先着率は、各営業日の高値・安値を日付順に見て先に到達した側を集計する。同一営業日に+5%と-3%へ両方到達した場合は順序不明として先着率の分母から除外する。

副項目として翌日、5日後、10日後、15日後の勝率を出力する。サマリでは合計損益を出力しない。

出力ファイル名は `backtest_a9r2_summary-*` と `backtest_a9r2_result-*` とする。
