# A11統合バックテスト

## 正式な起動方法

```powershell
python backtest_a11.py
```

正式入口は `backtest_a11.py` とする。

一度選択した監視銘柄ファイルは `.backtest_settings.json` に記憶する。次回起動時にファイルが存在すれば自動入力し、存在しない、または設定を読み込めない場合はファイル選択画面を表示する。

## GUI設定

- 開始日、終了日
- 監視銘柄Markdownファイル
- 出力先フォルダ
- 集計方法: 全シグナル法、ポジション法
- 25日乖離率の最低値、最高値
- エントリー条件
  - 日足: 翌営業日始値、前日終値
  - 日中足（5分足）: 11:00、14:00
- 日足条件
  - 3日間の安値切り下げ: 1回も許容しない、1回のみ許容、2回許容、考慮しない、連続切下(テスト用)
  - 終値位置: 考慮せず、30%以上、40%以上、50%以上、60%以上
  - 直下支持線距離: 考慮しない、0.7ATR超を除外、1.0ATR超を除外
  - 5日線傾き > 0 を条件にする
  - 5日線傾き鈍化: 考慮しない、前日・3日前とも許容しない、前日・3日前のいずれかのみ許容、3日前のみ許容、前日のみ許容
  - 25日線傾きの扱い: 傾き負を即除外、即除外しない、5日前より傾き鈍化除外、傾き鈍化・傾き負いずれも除外
- 日中足条件（5分足）
  - 崩れスコア除外: 考慮しない、3点以上、4点以上、5点以上
- `MA25条件割当` は、現在のフォームと同じエントリー条件で、25日線乖離率を
  `-4%超～-2%以下`、`-2%超～0%以下`、`0%超～2%以下`、`2%超～4%以下`、`4%超～6%以下`、
  `6%超～8%以下`、`8%超～10%以下`、`10%超～12%以下` にした8条件を実行キューへ追加する。

開始日の初期値は、実行日から60カレンダー日以内で一番早い営業日とする。終了日の初期値は直近の営業日とする。営業日の初期値計算は土日を除く平日基準で行う。日足エントリー、かつ崩れスコア除外を考慮しない日足のみの設定では、5分足取得制約による開始日制限をかけない。

安値切り下げは、直近3営業日それぞれの安値を前営業日と比較して数える。「1回も許容しない」「1回のみ許容」「2回許容」は、許容回数を超えた場合に除外する。「考慮しない」は条件を無効化する。「連続切下(テスト用)」は3営業日連続で安値を切り下げた場合に除外する。

日足の翌営業日始値エントリーでは、乖離判定日の翌営業日の日足始値を買値とする。日中足の11:00、14:00エントリーでは、乖離判定日の翌営業日の指定時刻直前までに確定した最後の5分足終値を買値とする。

位置条件は、日足条件では「終値位置」、日中足条件では「終端位置」と表示する。計算式は `(判定価格 - 安値) / (高値 - 安値) * 100` とする。日足エントリーでは、前日の日足高値～安値に対する終値位置を使う。11:00、14:00エントリーでは、エントリー判断時点までに確定した分足の高値～安値に対する終端位置を使う。位置条件を「考慮せず」にした場合、この条件では除外しない。

5日線傾き鈍化は、エントリー判定日の5日線傾きと、前営業日・3営業日前の5日線傾きを比較する。Aを「当日傾き < 前日傾き」、Bを「当日傾き < 3営業日前傾き」とする。「前日・3日前とも許容しない」はAまたはBがあれば除外する。「前日・3日前のいずれかのみ許容」はAとBのどちらか一方のみなら許容し、AとBが両方あれば除外する。「3日前のみ許容」はBのみ許容し、Aがあれば除外する。「前日のみ許容」はAのみ許容し、Bがあれば除外する。「考慮しない」はこの条件では除外しない。

崩れスコアは、VWAPなどの単独条件で除外する代わりに、5分足で判定する複数の弱さを合計して除外するための指標とする。VWAPは単独除外条件としては使わず、崩れスコアの1項目として扱う。翌営業日始値エントリーでは、始値時点で5分足の判定材料が確定していないため崩れスコア条件は使えない。

| 条件 | 点数 |
|---|---:|
| 現在値がVWAP未満 | +1 |
| 直近3営業日の安値切り上げがすべて不成立 | +1 |
| 直近3営業日の高値更新がすべて不成立 | +1 |
| 終端位置が40%未満 | +1 |
| 出来高20日平均比が100%以上、かつ陰線または上値失速 | +1 |

上値失速は「上ヒゲが当日値幅の45%以上、かつ実体の1.5倍以上」とする。式は `上ヒゲ比率 >= 45% かつ 上ヒゲ >= 実体 * 1.5` とする。

出来高20日平均比は、指定時刻までに確定した分足出来高の累計を寄り前時点の20日平均出来高で割って計算する。

直下支持線は、判定時点で既知の25日線、75日線、20日安値、60日安値のうち、現在値より下にある最も近い水準とする。ATRは寄り前時点のATR14を使う。直下支持線距離を考慮する場合、距離が選択したATR倍率を超える銘柄を単項目で除外する。崩れスコアには加点しない。

25日線傾きの扱いで「傾き負を即除外」を選んだ場合は従来どおりエントリー前に除外する。「即除外しない」を選んだ場合は25日線傾きでは除外せず、崩れスコアにも加点しない。「5日前より傾き鈍化除外」を選んだ場合は、エントリー判定時点の25日線傾きが5営業日前の25日線傾きより小さい場合に除外する。「傾き鈍化、傾き負いずれも除外」を選んだ場合は、25日線傾きが0%未満、または5営業日前より鈍化している場合に除外する。5営業日前の25日線傾きが計算できない場合は、鈍化判定を有効にした条件では除外する。

## エントリー条件

次の条件をすべて満たす場合にエントリーする。

- 前日と判定時点の25日乖離率が設定範囲内
- 25日線傾きの扱いに応じて、25日線が下向きでない、または5営業日前より鈍化していない
- 5日線傾き条件を有効にした場合、判定時点の5日線が前営業日の5日線より上
- 5日線傾き鈍化を考慮する場合、選択した鈍化許容ルールに抵触しない
- 安値切り下げ回数が除外基準未満、または安値切り下げ条件が考慮しない
- 終値位置または終端位置を考慮する場合、選択した閾値以上
- 直下支持線距離を考慮する場合、選択したATR倍率以下
- 崩れスコア除外を有効にした場合、崩れスコアが設定閾値未満

位置条件の計算に必要な高値・安値が取得できない場合、または高値と安値が同値でレンジが0の場合は、位置条件を計算できないためスキップする。

## ポジション法

全シグナル法は、同一銘柄が連続してエントリー条件を満たした場合も、各シグナルを独立した取引として集計する。ポジション法は、`-4%超～12%以下`の8つのMA25乖離率帯で成立した候補を統合し、銘柄コードごとにポジションを管理する。帯域が変わっても条件成立が続いている場合はリセットとみなさない。

5・10・15営業日の3種類は、それぞれ独立したポジションシミュレーションとして一度に計算する。

- `FLAT`: 未保有。シグナル成立時に新規エントリーする
- `OPEN`: 保有中。決済日を含む期間中の同一銘柄のシグナルは「保有中シグナル」として除外する
- `COOLDOWN`: 決済後。決済翌営業日以降に、全8帯域を通じてエントリー条件が1営業日以上不成立になるまで再エントリーしない
- 条件不成立を確認後も、前回エントリー日と新しいシグナル日の間に3営業日が完全に経過するまで再エントリーしない

エントリー後の翌営業日から日足の高値・安値を日付順に確認し、`+5%`に到達した場合はエントリー価格の105%、`-3%`に到達した場合は97%で決済する。同一営業日に両方へ到達した場合は、保守的に`-3%`が先に到達したものとする。どちらにも到達しない場合は、エントリーからX営業日後の終値で決済する。

各評価期間のサマリーに次を出力する。

- シグナル発生件数
- 独立エントリー件数
- 重複除外
- 対象銘柄数（独立エントリーが1件以上ある銘柄のユニーク数）
- 重複除外内訳: 保有中シグナル、シグナル未リセット、最低再エントリー間隔未満

`シグナル発生件数 = 独立エントリー件数 + 重複除外`、および`重複除外 = 3つの内訳の合計`が常に成立すること。「独立上昇波動数」は出力しない。

ポジション法は、3評価期間のサマリーと個別取引をまとめた単一ファイル `bt11_position_MA25(-4,12)-<実行日>_result-*`を出力する。

## 結果解析

主項目として5・10・15営業日の各期間について、次を集計する。

- +5%到達率
- -3%到達率
- 最大順行幅（MFE）中央値
- 最大逆行幅（MAE）中央値
- +5%先着率
- -3%先着率

+5%先着率と-3%先着率は、各営業日の高値・安値を日付順に見て先に到達した側を集計する。同一営業日に+5%と-3%へ両方到達した場合は順序不明として先着率の分母から除外する。

副項目として翌日、5日後、10日後、15日後の勝率を出力する。サマリでは合計損益を出力しない。

出力ファイル名は `bt11_MA25(<乖離率最低値>,<乖離率最高値>)-<実行日>_summary-*` と
`bt11_MA25(<乖離率最低値>,<乖離率最高値>)-<実行日>_result-*` とする。
