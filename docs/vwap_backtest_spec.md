# VWAP維持・25日線基準バックテスト仕様書

> この文書は旧VWAP単独版の履歴資料です。現在の正式仕様と起動方法は `docs/a8_backtest_spec.md` のA9r2仕様を参照してください。

## 1. 目的

yfinanceから取得した価格データを使い、監視銘柄について前日・当日の25日移動平均乖離率、25日線の傾き、VWAP維持を評価する。条件成立時の価格を買値とし、1、5、20営業日後の損益、および5、20営業日内の最大含み損を集計する。

既存のA7R2ドメインルールおよびファンダメンタル条件は使用・変更しない。

## 2. 起動方法

```powershell
python backtest_a9r2.py
```

起動すると専用GUIを表示する。

## 3. GUI入力

- 開始日、終了日: カレンダー選択
- 監視銘柄ファイル: Markdownファイル選択
- 出力先フォルダ
- 25日乖離率の最低値、最高値
- VWAP維持判定・エントリー時刻: `前日終値`、`11:00`、`14:00`

乖離率条件は次式とする。

```text
最低値 < 25日乖離率 <= 最高値
25日乖離率 = (終値 / 25日移動平均 - 1) * 100
```

## 4. 日付の定義

GUIの開始日・終了日は「25日乖離率を判定する日」の範囲である。この日を乖離判定日と呼ぶ。

- `前日終値`: 乖離判定日の終値でエントリーする。
- `11:00`、`14:00`: 乖離判定日の翌営業日の指定時刻にエントリーする。

11時・14時から見た場合、乖離判定日の終値が「前日終値」に該当する。

## 5. VWAP、25日線とエントリー条件

まず、乖離判定日の終値について25日乖離率が基準範囲内であることを確認する。

11:00、14:00判定では、翌営業日の指定時刻直前までに確定した最後の5分足終値を判定価格兼買値とする。yfinanceの分足時刻はバー開始時刻として扱うため、11:00判定では10:55開始足、14:00判定では13:55開始足を使う。

当日の暫定25日線は、前日までの直近24営業日の終値と判定価格から計算する。未来の当日終値は使用しない。

```text
当日暫定25日線 = (前日までの直近24終値の合計 + 判定価格) / 25
当日25日乖離率 = (判定価格 / 当日暫定25日線 - 1) * 100
25日線傾き = (当日暫定25日線 / 前日25日線 - 1) * 100
```

指定時刻までの確定済み5分足から累積VWAPを計算する。

```text
代表価格 = (高値 + 安値 + 終値) / 3
累積VWAP = Σ(代表価格 * 出来高) / Σ出来高
```

次の4条件をすべて満たす場合にエントリーする。

- 最低値 < 前日25日乖離率 <= 最高値
- 最低値 < 当日25日乖離率 <= 最高値
- 25日線傾き >= 0%（横ばい以上）
- 判定価格 >= 累積VWAP

## 6. 売却と損益

実際のエントリー日を基準に、1、5、20営業日後の終値で1株を売却したものとして計算する。

```text
損益額 = 売値 - 買値
損益率 = (売値 / 買値 - 1) * 100
```

手数料、税金、スリッページ、配当は考慮しない。同一銘柄が複数日に条件成立した場合、それぞれ独立した取引として扱い、重複保有を許可する。

## 7. 最大含み損

エントリー翌営業日から5営業日目、20営業日目までの日足安値の最低値を使う。エントリー当日は対象に含めない。

```text
最大含み損額 = min(0, 期間内最低値 - 買値)
最大含み損率 = min(0, (期間内最低値 / 買値 - 1) * 100)
```

期間中の安値が買値を下回らなかった場合、最大含み損は0とする。

## 8. 勝率とサマリー

各売却期間について、将来価格を取得できたエントリーだけを分母とする。

```text
勝率 = 損益額が0より大きい件数 / 評価可能件数 * 100
```

損益ゼロは勝ちに含めない。サマリーには以下を記載する。

- 監視銘柄数、評価件数、エントリー件数
- 1、5、20営業日後の評価可能件数
- 勝率、平均損益率
- 5営業日後リターン: 5営業日後損益率の平均
- 最大含み損 平均・中央値: 5営業日内の最大含み損率の平均・中央値
- -3%逆行率: 5営業日内の最大含み損率が-3.00%以下となった取引の割合
- 前日・当日の条件範囲外、25日線下向き、VWAP未維持、データ不足などのスキップ件数

最大含み損関連の指標は、5営業日分の将来データが揃う取引だけを分母とする。

## 9. 出力

出力先フォルダにUTF-8のMarkdownを作成する。

- `backtest_Summary-MM-DD-HH-mm.md`: 条件、件数、勝率、平均損益
- `backtest_result--MM-DD-HH-mm.md`: 個々のエントリーと将来損益、最大含み損

同一分に複数回実行した場合は同名ファイルを上書きする。

## 10. データ制約と欠損

- 日足: `interval="1d"`、自動調整後OHLCを利用する。
- 分足: `interval="5m"`、自動調整後OHLCを利用する。
- yfinanceの分足取得制約により、GUIでは実行日の59日前以降を開始日に指定する。
- 終了日に未来日は指定できない。
- 25日移動平均、翌営業日、指定時刻分足、将来価格がない場合は該当評価をスキップまたは`N/A`とする。
- yfinanceは非公式APIを利用するため、データ欠損や取得制限によって再現結果が変わる可能性がある。
